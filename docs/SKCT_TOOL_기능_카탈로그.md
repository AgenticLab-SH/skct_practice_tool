# SKCT Tool 기능 카탈로그
작성일시: 2026-04-10 09:48:25 +09:00

이 문서는 지금 기준으로 사용자가 실제로 쓸 수 있는 기능과 운영자가 관리할 수 있는 기능을 짧고 안정적으로 설명하는 기준서입니다. 날짜별 작업 과정은 `docs/agent/worklog/`에 남기고, 이 문서는 현재 기능의 기준 상태만 유지합니다.

## 1. 이 문서의 역할
- 새 기능 추가, 기존 기능 수정, 기능 제거가 있으면 같은 턴에 이 문서를 갱신합니다.
- “지금 무엇이 되는지”를 확인하는 문서이므로, 오래된 시도나 실패 이력은 적지 않습니다.
- 기능 설명이 바뀌지 않은 내부 리팩터링이라면 이 문서를 억지로 수정하지 않고, 작업 로그에 `기능 문서 변경 없음`만 남깁니다.

## 2. 일반 사용자 핵심 기능

### 2.1 메인 연습 작업공간
- OMR 마킹, 정답 입력, 채점, 통계 확인이 가능합니다.
- 전체 타이머와 과목별 시간 흐름을 기준값에 맞춰 사용합니다.
- 쉬는 시간 건너뛰기와 문항 흐름 보정이 포함됩니다.

### 2.2 보조 도구와 화면 구성
- 계산기, 메모장, 그림판을 메인 화면과 팝업 모드에서 사용할 수 있습니다.
- 메모장은 일반 텍스트 입력 커서를 유지하고, 그림판은 원 없는 얇은 십자 형태의 그리기 커서를 사용합니다.
- 팝업 창 크기, 위치, OMR 폭, 하단 여백, 우측 버튼 열 등은 비율 기반 기본값 구조를 따릅니다.
- 팝업 창은 너무 좁게 열리지 않도록 최소 폭 보정이 들어가며, 기본 팝업 폭에서도 OMR 입력칸과 하단 버튼이 함께 보이도록 맞춥니다.
- `더보기`, `가이드`, `고급 기능` 버튼이 역할별로 분리되어 있습니다.
- `더보기`에서는 활성 세션, 커뮤니티, 운영 후원과 함께, 핵심 연습 도구와 분리된 `별도 테스트 자료` 안내 페이지로 이동할 수 있습니다.
- `가이드`는 일반 모드 흐름만 설명하고, 고급 모드 내용은 `고급 안내`와 `고급 활용`로 분리해 연결합니다.
- 설정, 더보기, 활성 세션, 과목별 상세 통계, 정오표 일괄입력 같은 버튼 내부 화면은 긴 설명을 기본 화면에 계속 노출하지 않고, 헤더의 `?` 빠른 도움말로 다시 볼 수 있습니다.

### 2.3 문서형 페이지
- `guide`, `faq`, `pricing`, `privacy`, `terms` 페이지를 별도 경로로 제공합니다.
- 이 페이지들은 메인 연습 도구와 정책/설명/가격 안내를 섞지 않기 위한 용도입니다.

### 2.4 신청, 조회, 로그인 관련 기본 흐름
- 사용자 신청/조회 흐름은 `이메일` 기준으로 안내합니다.
- 신청 상태 조회는 `신청 이메일 + 조회 비밀번호` 기준으로 동작합니다.
- 고급 모드 로그인은 `신청 이메일 또는 로그인 ID + 비밀번호` 기준으로 동작합니다.
- 신청번호는 내부 식별과 관리자 작업용으로만 남기고, 사용자 로그인 식별자로는 쓰지 않습니다.
- `고급 안내`는 신청과 승인 후 진입까지 설명하고, `고급 활용`은 이미 들어온 뒤 어떤 버튼을 쓰는지만 설명합니다.

## 3. 고급 모드와 자료 보관함

### 3.1 고급 모드
- 일반 화면에는 `고급 기능` 버튼만 보이고, 실제 고급 기능은 유효한 고급 라이선스가 확인된 뒤에만 열립니다.
- 고급 모드에서는 일반 모드보다 더 많은 통계, 정오표 일괄입력, 추가 제어 기능이 열립니다.
- 일반 모드와 고급 모드의 노출 기능은 같지 않으며, 버튼 숨김뿐 아니라 실제 접근 경계도 다르게 유지합니다.
- 고급 모드 상단에는 현재 로그인 ID, 만료 시각, 자료 보관함 가능 여부, 우측 실제환경 여백 상태를 보여주는 상태 바가 나타납니다.
- OMR 하단의 `고급 복기 순서`는 기본 노출을 `?` 버튼 하나로 줄였고, 누르면 `정답 입력 -> 채점 -> 복기 버튼` 흐름을 빠른 도움말로 볼 수 있습니다.
- 고급 모드 우측 상단 말풍선 버튼은 placeholder가 아니라 `고급 활용` 모달을 바로 여는 빠른 진입 버튼으로 동작합니다.
- 고급 모드 안의 `전체 안내` 버튼에서는 신청 절차를 반복하지 않고, 이미지 대신 도식 카드로 `상단 상태 바`와 `OMR 아래 복기 구역`만 빠르게 보여줍니다.
- 기본 팝업 폭 기준에서도 고급 OMR의 5지선다 입력칸과 하단 버튼이 같이 보이도록 폭과 반응형 레이아웃을 다시 맞췄습니다.
- `고급 안내`와 `고급 활용` 화면은 기본 문구를 짧게 유지하고, 겹치는 설명은 `?` 빠른 도움말로 분리했습니다.
- 공개 배포에서도 고급 팝업 진입은 `advanced-tools.html` 인증 페이지를 통해 유지됩니다.

### 3.2 자료 보관함
- `study-archive.html`은 고급 모드 전용 보조 기능입니다.
- 일반 모드에서는 진입 버튼을 숨기고, 직접 URL 접근도 고급 라이선스 조건으로 다시 막습니다.
- `더보기 > 자료 보관함`은 새 탭으로 열리고, 현재 연습 페이지는 그대로 유지됩니다.
- 자료 보관함 안에서는 Firebase Auth 로그인 사용자만 자기 자료를 읽고 수정할 수 있습니다.

## 4. 관리자와 운영 기능
- 관리자 페이지에서 사이트 문구, 기본값, 후원/공지, 구독/라이선스 관련 값을 관리합니다.
- 관리자 페이지는 공개 배포 주소가 아니라 로컬 [admin.html](/C:/dev/01_career/_assets/tools/skct_tool/admin.html)로 열어야 하며, Windows에서는 [open_local_admin.cmd](/C:/dev/01_career/_assets/tools/skct_tool/scripts/open_local_admin.cmd) 또는 [open_local_admin.ps1](/C:/dev/01_career/_assets/tools/skct_tool/scripts/open_local_admin.ps1)로 한 번에 열 수 있습니다.
- VS Code에서는 `.vscode/tasks.json`의 `SKCT: 로컬 관리자 페이지 열기`, `SKCT: 로컬 관리자 페이지 종료` 작업으로 바로 실행할 수 있습니다.
- VS Code 작업은 PowerShell shell task가 아니라 `cmd.exe /c` process task라서 `-Command` 오류 없이 바로 실행됩니다.
- 수동 이용권 설정 안에서 `보안 API 기본 URL`을 저장해, 신청 저장/조회/고급 라이선스 확인을 나중에 서버 경유로 전환할 준비를 해둘 수 있습니다.
- `공지 관리`에서 저장한 Help/사이드바 공지는 이미 열려 있는 메인 페이지에도 새로고침 없이 다시 반영됩니다.
- 관리자 저장이 권한 또는 네트워크 문제로 실패하면, 관리자 화면에서 실패 이유를 바로 토스트로 안내합니다.
- 관리자 대시보드는 이제 Firebase Auth 사용자 객체만이 아니라 실제 토큰 확인까지 통과한 뒤에만 진입하고, 쓰기 작업도 저장 직전에 토큰을 다시 확인한 뒤 실행합니다.
- 관리자 저장이 `permission-denied`로 한 번 막히면, 토큰을 새로 확인하고 RTDB 연결을 다시 붙인 뒤 자동으로 한 번 더 저장을 시도합니다.
- 운영 텍스트의 최상위 기준은 Firebase RTDB `config/siteTextConfig`입니다.
- 기본값/레이아웃/문구는 코드 하드코딩만 바꾸지 말고 관리자 저장 흐름과 함께 맞춰야 합니다.

## 5. 기능 경계상 특히 중요한 기준
- 일반 모드와 고급 모드의 기능 경계가 무너지면 안 됩니다.
- 사용자 조회용 식별자와 실제 로그인 식별자를 섞으면 안 됩니다.
- 새 기능은 기존 메인 흐름을 덮어쓰지 말고, 별도 경로나 별도 권한 경계가 필요하면 분리합니다.
- 운영 반영 전에는 항상 로컬 또는 스테이징 기준으로 먼저 검증합니다.
