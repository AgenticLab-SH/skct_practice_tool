# 관리자 문구 편집기 확장
작성일시: 2026-04-09 02:42:28 KST

관리자 문구 편집기 확장 항목은 이번 라운드에서 로컬 코드와 문서 기준으로 완료했습니다.

## 2026-04-09 처리 결과
- `site-text-config.js` 카탈로그에 메인 사이드바, 유틸리티 모달, 지표 모달, 고급 안내, 고급 기능 관련 문구를 추가했습니다.
- 관리자 화면에 `전체`, `공통`, `고급`, `운영` scope 필터를 넣어 운영자가 문구 범위를 먼저 좁혀 볼 수 있게 했습니다.
- 선택한 문구의 범주와 실제 노출 위치를 보여 주는 힌트를 추가했습니다.
- 예전 운영 기본값이 새 코드 기본값을 덮어쓰지 않도록, 레거시 기본값 자동 보정 로직을 넣었습니다.

## 이번에 보존한 기준
- `config/siteTextConfig` 저장 흐름
- iframe 미리보기, 클릭 선택, 검색, 즉시 반영 흐름
- `supportConfig`와 `siteTextConfig`의 역할 분리
- sanitize 기반 HTML 렌더링 경로

## 로컬 검증
- 메인 페이지에서 새 카탈로그 문구가 실제 화면에 반영되는지 확인했습니다.
- 관리자 페이지 로딩 후 `siteTextScopeFilter`, `siteTextSelectedHint`, 검색 입력이 모두 존재하는지 확인했습니다.
- 관리자 페이지와 메인 페이지 모두 콘솔 `error` 0건을 확인했습니다.

## 남은 운영 후속
- 운영 Firebase `config/siteTextConfig`를 새 기본 문구로 공식 동기화할지는 사용자 승인 후 진행해야 합니다.
- 운영 관리자가 실제로 자주 수정하는 문구 묶음이 더 생기면 같은 방식으로 카탈로그를 계속 확장하면 됩니다.
