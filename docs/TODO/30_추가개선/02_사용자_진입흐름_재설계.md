# 사용자 진입흐름 재설계
작성일시: 2026-04-09 16:10:35 KST

사용자 진입흐름 재설계 항목은 이번 라운드에서 로컬 코드 기준으로 완료했습니다.

## 2026-04-09 처리 결과
- 일반 사용자 흐름은 `고급 안내 -> 신청 또는 조회 -> 고급 열기` 중심으로 단계가 보이게 다시 정리했습니다.
- 고급 사용자 흐름은 `고급 기능 -> 복기와 활용 예시` 중심으로 분리해, 일반 안내와 같은 설명을 반복하지 않게 했습니다.
- 일반 도움말도 새 사이드바와 `더보기` 구조 기준으로 다시 설명해, 첫 진입 사용자가 현재 단계와 다음 행동을 더 쉽게 이해하도록 맞췄습니다.
- 기존 이메일 중심 조회 흐름과 레거시 조회 호환성은 코드상 그대로 유지했습니다.

## 이번에 보존한 기준
- 신청 저장 직후 조회 입력이 이어지는 흐름
- 저장된 라이선스 재확인과 고급 모드 진입
- 고급 기능 모달의 복기 기능 안내
- 레거시 조회 호환 경로

## 로컬 검증
- 메인 페이지에서 일반 안내와 고급 기능 모달이 정상적으로 열리고, 새 문구가 보이는지 확인했습니다.
- 신청과 조회 관련 입력 영역이 삭제되지 않았는지 확인했습니다.
- 새 구조 적용 후 콘솔 `error` 0건을 확인했습니다.

## 현재 남은 후속
- 실제 사용자의 문의 패턴을 보면, 문장 길이만 더 줄일지 여부를 판단하면 됩니다.
- 현재는 구조보다 설명의 순서를 정리한 단계이므로, 추가로 시각적 진행 표시가 필요하면 후속 개선으로 붙일 수 있습니다.
