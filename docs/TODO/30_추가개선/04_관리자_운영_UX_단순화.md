# 관리자 운영 UX 단순화
작성일시: 2026-04-09 02:42:28 KST

관리자 운영 UX 단순화 항목은 이번 라운드에서 로컬 코드 기준으로 완료했습니다.

## 2026-04-09 처리 결과
- 사이트 텍스트 편집기에서 문구를 `전체`, `공통`, `고급`, `운영` 범위로 나눠 볼 수 있게 했습니다.
- 현재 선택한 문구가 어떤 범주이고 어디에 보이는지 바로 읽을 수 있는 힌트를 추가했습니다.
- 긴 문구 편집에서 `무엇을 바꾸는지`를 먼저 이해하게 해, 운영자가 저장 영향 범위를 더 쉽게 판단하도록 정리했습니다.

## 이번에 보존한 기준
- 개별 저장 버튼과 전체 저장 버튼
- 수동 신청 백업, 복원
- 키 폴더 연결과 서명키, 복호화키 관리
- 고급 구독 저장 시 실제 로그인 레코드 동기화 흐름

## 로컬 검증
- 관리자 페이지 로딩 후 scope 필터와 선택 힌트가 모두 존재하는지 확인했습니다.
- 관리자 페이지 콘솔 `error` 0건을 확인했습니다.
- 기존 로그인 화면과 편집기 DOM 구조가 깨지지 않았는지 확인했습니다.

## 남은 운영 후속
- 실제 운영자 사용 패턴을 기준으로, 이후에는 `자주 하는 일`과 `가끔 하는 일` 카드 구분을 추가할 수 있습니다.
- 이번 라운드에서는 저장 로직을 건드리지 않고 판단 보조 정보만 먼저 강화했습니다.
