# 그림판 커서 가시성 문제
작성일시: 2026-04-09 16:10:35 KST

그림판 커서 가시성 문제는 이번 라운드에서 로컬 코드 기준으로 해결했습니다.

## 2026-04-09 처리 결과
- 캔버스 위에 전용 커서 인디케이터를 추가하고, 기본 브라우저 커서는 캔버스 영역에서 숨기도록 바꿨습니다.
- 커서 인디케이터는 원형 링과 십자선을 함께 그려 밝은 배경에서도 대비가 유지되게 했습니다.
- 마우스 이동 시에만 표시하고, 터치 입력이나 메모장 탭 전환 시에는 숨기도록 처리했습니다.
- 오버레이는 `pointer-events: none`으로 두어 기존 그리기 이벤트를 막지 않게 했습니다.

## 이번에 보존한 기준
- 메모장, 그림판 탭 전환
- 캔버스 resize 시 기존 그림 보존
- 마우스와 터치 입력 동시 지원
- 선 굵기 설정과 `삭제` 버튼 동작

## 로컬 검증
- 메인 페이지에서 그림판 탭 전환 후 캔버스 진입 시 커서 인디케이터가 생성되는지 확인했습니다.
- 마우스 이동 시 커서 인디케이터가 표시되고 좌표를 따라 움직이는지 확인했습니다.
- 탭 전환과 터치 입력 경로에서 커서가 숨겨져 기존 입력을 방해하지 않는지 확인했습니다.

## 현재 남은 후속
- 실제 사용하는 브라우저 조합에서 커서 배율 차이가 체감되면 CSS 크기만 미세조정하면 됩니다.
