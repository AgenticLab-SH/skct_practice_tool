1. 문서의 문제 정의와 최근 처리 결과를 함께 읽습니다.

5. 작업 결과를 `docs/agent/worklog/`에 남깁니다.

- 완료는 `구현함`이 아니라 `검증 체크리스트 통과 + 기존 기능 보존 근거 확보 + 기록 완료`까지 포함합니다.
- 운영 반영이 남아 있으면 문서 안에 `남은 운영 후속`을 따로 적습니다.
- 운영 반영까지 끝났다면, `남은 운영 후속`에는 배포 대기 문구 대신 실제로 남은 모니터링/추가 검증만 남깁니다.
