# 에이전트 전달 프롬프트
작성일시: 2026-04-09 16:10:35 KST

이 문서는 새 TODO가 다시 생겼을 때 바로 복사해 쓸 수 있는 프롬프트와, 운영 후속 전용 프롬프트를 보관합니다.

## 현재 상태
- 2026-04-09 라운드 기준 현재 작업 큐는 모두 완료입니다.
- 운영 배포와 주요 rules 반영도 대부분 끝났습니다.
- 따라서 아래 프롬프트는 `새 TODO가 추가되거나`, `실계정 검증/모니터링/보안 확장` 같은 후속을 이어갈 때만 다시 사용합니다.

## 새 TODO 시작 프롬프트
```text
docs/TODO/00_TODO_INDEX.md, docs/TODO/01_AGENT_RUNBOOK.md, docs/TODO/02_AGENT_WORK_QUEUE.md를 먼저 읽고 시작하세요.
작업은 docs/TODO/02_AGENT_WORK_QUEUE.md의 순서대로 한 번에 하나씩만 진행하세요.
각 TODO 문서에 적힌 보존 조건, 위험 구간, 최근 처리 결과, 검증 체크리스트를 먼저 확인한 뒤 구현하세요.
현재 문서의 완료 조건을 만족하고 검증 체크리스트를 통과하기 전에는 다음 문서로 넘어가지 마세요.
각 작업이 끝날 때마다 docs/agent/worklog에 기록을 남기고, 35_LEARNING_NOTES.md도 갱신하세요.
기존 기능 보존과 새 요구가 충돌하거나 운영 데이터 없이는 검증할 수 없으면 멈추고 보고하세요.
```

## 운영 후속 전용 프롬프트
```text
docs/TODO/00_TODO_INDEX.md와 70_USER_TODO.md를 먼저 읽고, 이번 라운드에서 남긴 운영 후속만 처리하세요.
사용자 승인 없는 운영 반영은 하지 말고, 스테이징 검증과 운영 반영은 분리하세요.
이미 배포된 항목을 다시 배포 대기로 되돌리지 말고, 실제로 남은 실계정 검증, 운영 모니터링, 보안 확장 검토만 다루세요.
결과는 docs/agent/worklog에 별도 기록하세요.
```

## 사용 메모
- 새 구현 작업이면 `새 TODO 시작 프롬프트`를 씁니다.
- 배포, rules 반영, 운영 검증이면 `운영 후속 전용 프롬프트`를 씁니다.
