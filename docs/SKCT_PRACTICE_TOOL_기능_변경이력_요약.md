# SKCT Tool 기능 변경이력 요약
작성일시: 2026-04-06 14:44:17 KST

이 문서는 지금까지 진행한 큰 변경들을 날짜 흐름대로 요약한 문서입니다.  
세부 코드나 세세한 검증 로그는 `docs/agent/worklog/`에 있고, 이 문서는 “무엇이 달라졌는지”를 빠르게 파악하는 용도입니다.

## 1. 운영/배포 프로세스 정리

- 운영을 테스트 서버처럼 쓰지 않도록, `로컬 -> 스테이징 -> 승인 -> 운영` 순서를 고정했습니다.
- 스테이징을 별도 경로로 분리하고, 운영 반영 전에 먼저 원격 확인하는 흐름을 만들었습니다.

관련 로그:
- [2026-04-05_release_process_design.md](/C:/dev/01_career/_assets/tools/skct_practice_tool/docs/agent/worklog/2026-04-05_release_process_design.md)

## 2. 스테이징 읽기 전용 구조 도입

- 운영 데이터를 그대로 읽되, 스테이징에서는 방문/좋아요/댓글/설정 저장이 차단되도록 정리했습니다.
- 관리자 게이트를 통과해야만 스테이징 테스트 페이지에 들어가도록 바꿨습니다.

관련 로그:
- [2026-04-05_staging_readonly_and_exam_mode_updates.md](/C:/dev/01_career/_assets/tools/skct_practice_tool/docs/agent/worklog/2026-04-05_staging_readonly_and_exam_mode_updates.md)

## 3. 팝업 기본값 편집기와 비율 기반 저장

- 팝업 창 크기, 위치, OMR 폭, 하단 여백, 우측 버튼 열, 메모 글씨 크기, 그림판 굵기를 비율값으로 다루게 했습니다.
- 개발자 페이지에서만 기본값 편집/저장이 가능하도록 분리했습니다.
- 테스트 저장과 운영 반영을 3단계 흐름으로 나눴습니다.

- [2026-04-05_staging_popup_ui_and_calc_alignment.md](/C:/dev/01_career/_assets/tools/skct_practice_tool/docs/agent/worklog/2026-04-05_staging_popup_ui_and_calc_alignment.md)

## 4. 팝업 UI 실전 유사화

- 우측 상단 채팅 버튼, 우측 하단 물음표 버튼 자리 구현
- 메모장/그림판/삭제 탭 구조 정리
- 계산기/메모장/그림판 레이아웃을 실제 화면에 더 가깝게 맞춤
- 우측 버튼 열과 하단 여백을 별도 비율로 분리

## 5. 계산기 대규모 개선

- 실전형 키 배열 정리
- `=`를 눌렀을 때만 계산
- 사칙연산 우선순위 반영
- 이전 계산 결과를 이어서 계산 가능
- 최근 계산 이력 표시 개선
- 계산기 영역 클릭 시 포커스 이동

## 6. OMR 동작 개선

- 문항 번호 정렬 보정
- 선택지 간격 조정
- 이전 문항 재선택 허용
- 실전/연습 모드별 입력 제한 분리
- 건너뛴 문항의 정답 입력 가능 안내 보강

## 7. 타이머와 쉬는 시간 로직 정리

- 전체 시간 75분 기준 재정리
- 4분 조기 종료처럼 보이던 문제 보정
- 쉬는 시간 건너뛰기 버튼 추가
- 문항 건너뛰기 시 문항별 시간 초기화 보정

## 8. 후원/공지/통계/운영 문구 관리자화

- 후원 모달 문구, 쉬는 시간 문구, 문의 링크, 후원 버튼 문구를 관리자 페이지에서 저장 가능하게 했습니다.
- 후원 목록 위치를 `커피 후원` 모달 상단으로 이동했습니다.
- 후원 목록 전환 속도도 관리자 페이지에서 조절 가능하게 했습니다.

## 9. 고급모드 도입

- 숨김 진입 방식과 고급모드 팝업 전환 구조를 만들었습니다.
- 일반모드와 고급모드 기능을 분리했습니다.
- 고급 기능 안내 모달과 고급 전용 기능 모달을 나눴습니다.
- 일반모드에는 안내만, 고급모드에는 실제 확장 기능이 보이도록 정리했습니다.

관련 문서:

## 10. 고급 구독 관리 방향 전환

- 평문 비밀번호 목록 방식에서, 구독 레코드 기반 구조로 바꾸기 시작했습니다.
- 개발자 페이지에서 표 형태로 구독 상태, 만료일, 외부 ID, 비밀번호를 관리할 수 있게 정리했습니다.
- 비밀번호는 평문 저장 대신 `salt + hash` 기반으로 다루도록 보강했습니다.
- 최근에는 스테이징에서 `고급 안내` 모달에서 바로 비밀번호 입력 후 고급모드 진입이 가능하도록 다듬었습니다.

## 11. 토스 결제 분리 프로젝트 설계

- 고급 구독을 결제형으로 바꾸기 위해 토스 결제 기반 분리 프로젝트 설계서를 작성했습니다.
- 아직 구현은 하지 않았고, 구조/보안/데이터 모델만 정리된 상태입니다.

## 12. 지금 기준으로 중요한 상태

- 일반 기능과 고급 기능은 이미 분기되어 있습니다.
- 스테이징은 운영과 분리된 검증 경로가 있습니다.
- 고급 구독은 스테이징에서 표 기반으로 관리되기 시작했습니다.
- 토스 결제는 아직 설계 단계이며, 별도 프로젝트로 진행할 예정입니다.
