# 2026-04-11 고급모드 정오표 일괄입력 클릭 바인딩 보강

## 요청
- 고급모드에서 `정오표 일괄입력`이 동작하지 않는다는 사용자 제보 기준으로 점검 및 빠른 수정

## 확인한 내용
- 로컬 기준 `index.html` / `main.js`에서는 버튼, 모달, 파서, 일괄 반영 로직이 모두 존재했습니다.
- 고급 상태를 강제로 복원한 로컬 브라우저 재현에서는 버튼 클릭 -> 모달 열림 -> 정오표 분석 -> 정답 일괄 반영까지 정상 동작했습니다.
- 다만 클릭 연결이 해당 DOM 노드에 대한 직접 바인딩만 사용하고 있어, 팝업/재렌더/캐시 혼선이 있는 환경에서는 사용자가 “버튼이 안 먹는다”로 체감할 여지가 있었습니다.
- 메인 운영 파일과 스테이징 사본의 버튼 라벨도 서로 달라 사용자가 기능명을 찾기 어렵습니다.

## 적용한 수정
- `main.js`
  - `bulkCorrectImportBtn`, `bulkCorrectImportParseBtn`, `bulkCorrectImportApplyBtn`를 직접 바인딩 대신 `id` 기준 delegated click 바인딩으로 보강
- `index.html`
  - 버튼 문구를 `📥 정오표 일괄입력`으로 통일
  - 버튼에 `type="button"` 명시
- `staging/site/index.html`
  - 동일하게 `type="button"` 명시
- `staging/site/assets/scripts/app.bundle.js`
  - 메인과 동일한 delegated click 바인딩 반영

## 검증
- 로컬 서버 `http://127.0.0.1:8140`
- Playwright로 고급 팝업 상태를 강제로 복원한 뒤 아래 흐름 확인
  - OMR 열기
  - 정답 입력 모드 전환
  - `정오표 일괄입력` 버튼 클릭
  - 예시 표 붙여넣기
  - `붙여넣기 분석`
  - `정답 일괄 반영`
- 확인 결과
  - 분석 요약: `데이터 줄 2개 중 2문항을 정답으로 읽었습니다.`
  - 반영 상태: `2문항의 정답을 반영했습니다.`
  - OMR 첫 2문항 정답 표시도 각각 `2`, `3`으로 반영 확인
