# 선택 배포 항목 정리
작성일시: 2026-04-09 23:49:44 +09:00

## 1. 질문

- 사용자가 커스텀 도메인과 광고를 모르는 상태에서, 이 항목들을 빼고 배포하면 문제가 되는지 물었습니다.

## 2. 결론

- 커스텀 도메인과 광고는 현재 배포의 필수 조건이 아닙니다.
- 둘 다 없이 먼저 배포해도 서비스 자체에는 문제가 없습니다.
- 지금 배포에서 중요한 것은 `도메인/광고`가 아니라 `민감 흐름 서버 분리 완료 여부`입니다.

## 3. 판단 근거

- GitHub Pages는 기본 `github.io` 도메인으로도 정상 운영할 수 있습니다.
- 커스텀 도메인은 브랜드/신뢰/공유 편의성 개선용이지, 사이트 오픈 자체의 필수 조건은 아닙니다.
- 광고는 승인, 정책 검토, 코드 삽입, 트래픽/콘텐츠 적합성 판단이 필요한 후행 작업입니다.
- 이 프로젝트 기준으로는 광고보다 먼저 사이트 구조 안정화와 보안 전환이 우선입니다.

## 4. 사용자 안내 방향

- 지금 당장은 `public-clean` 기준 공개 배포와 보안 API 전환 여부만 신경 쓰면 됩니다.
- 커스텀 도메인은 나중에 붙여도 됩니다.
- 광고도 나중에 붙여도 됩니다.
- 초보자 기준으로는 `기본 github.io 주소로 먼저 운영 -> 이후 도메인 구매 -> 마지막에 광고 검토` 순서가 가장 안전합니다.

## 5. 기능 문서 변경 여부

- 기능 문서 변경 없음
