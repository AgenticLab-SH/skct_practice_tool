# 일반·고급 안내 흐름 통합 정리
작성일시: 2026-04-10 12:55:27 +09:00

## 1. 요청

- 일반 모드와 고급 모드의 안내 메시지, 버튼 내부 메시지를 전체적으로 다시 보고 하나의 흐름으로 정리한다.
- 서로 겹치는 설명은 줄이고, 불필요한 문장은 과감히 제거한다.
- 기능 설명은 기본 노출에서 접을 수 있게 바꾸고, `?` 버튼을 적극 활용해 공간을 최적화한다.

## 2. 수행 내용

- `index.html`
  - 일반 도움말 모달 상단에 `일반 흐름` 요약 스트립을 추가하고, 예시 이미지와 참고 블록은 `details` 접힘 구조로 옮겼다.
  - 고급 신청 모달 상단에 `신청 -> 승인 확인 -> 진입` 흐름 스트립을 추가하고, 기능 카드 상세와 비밀번호 기준 설명을 접힘 구조로 줄였다.
  - 고급 활용 모달 상단에 `상태 확인 -> OMR 복기 -> 자료 보관함` 요약 스트립을 두고, 예전 이미지 설명은 접힌 도식 보기로 이동했다.
  - 고급 OMR 하단 설명은 `고급 복기 / 정답 입력 -> 채점 -> 복기 버튼 / 전체 안내 / ?` 형태의 짧은 레이아웃으로 바꿨다.

- `main.css`
  - `guide-flow-strip`, `guide-fold` 같은 공통 스타일을 추가해 일반/고급 안내 화면 모두 같은 정보 구조를 쓰도록 맞췄다.
  - 팝업 모드와 고급 모드의 좁은 폭에서도 요약 줄과 버튼이 먼저 보이도록 간격과 정렬을 조정했다.

- `main.js`
  - 빠른 도움말(`?`)의 context content를 일반 설정, 더보기, 고급 진입, 고급 OMR, 고급 활용 기준으로 다시 줄이고 역할을 분리했다.
  - 고급 상태 바와 OMR 복기 안내 fallback 문구도 짧게 바꿔, 기본 화면에서 같은 설명이 반복되지 않게 했다.

- `site-text-config.js`
  - Firebase `config/siteTextConfig`가 예전 문구를 다시 덮어쓰지 않도록 기본값과 `LEGACY_SITE_TEXT_DEFAULTS`를 함께 갱신했다.
  - 일반 도움말, 고급 신청, 고급 활용, 수동 신청 성공 메시지 등 실제 운영 문구가 새 구조로 migration 되게 맞췄다.

## 3. 검증

- `node --check main.js`
- `node --check site-text-config.js`
- 로컬 정적 서버에서 일반 도움말, 고급 신청, 고급 활용, 고급 OMR `?` 빠른 도움말을 열어 기본 노출과 접힘 구성이 의도대로 보이는지 확인했다.
- 고급 모드 강제 상태에서 OMR 입력칸과 하단 버튼이 요약 문구에 가려지지 않는지 함께 확인했다.

## 4. 판단 및 비고

- 이번 턴은 사용자 체감 안내 구조가 바뀌는 작업이므로 기능 기준서와 에이전트 기준서를 함께 갱신했다.
- 운영 배포는 이번 턴 요청에 포함되지 않아 진행하지 않았다. 현재 변경은 로컬 작업 브랜치 기준이다.
