# 2026-04-08 관리자 워크플로 재배치 및 구독 운영 복구
작성일시: 2026-04-08 22:58:07 KST

## 작업 배경
사용자는 관리자 페이지에서 다음 문제를 동시에 겪고 있었다.

1. 신청 복호화 키와 라이선스 서명 키의 역할이 화면상으로 충분히 설명되지 않았다.
2. 수동 구독 신청 목록에서 `승인/반려`만 가능하고 `차단/삭제`가 되지 않았다.
3. 예전에 쓰던 고급 계정 관리 표가 사실상 비활성화되어 있었다.
4. 관리자 페이지가 길어지면서 실제 운영 순서와 화면 순서가 어긋나 있었다.

## 이번에 반영한 코드 변경
### 1. 수동 신청 관리 강화
*   신청 카드에 `차단/차단 해제`와 `삭제` 동작을 추가했다.
*   차단/해제는 암호화된 신청 본문을 다시 저장하면서 상태와 관리자 메모를 함께 남기도록 했다.
*   삭제는 `/subscriptionRequests/{requestId}`를 Firebase에서 실제 제거하도록 했다.

### 2. 고급 구독 내역 복구
*   기존 레거시 문구를 `고급 구독 내역`으로 바꾸고 관리자 CRUD 화면으로 다시 활성화했다.
*   `config/advancedFeatureConfig`를 다시 읽고 저장하도록 복구했다.
*   수동 신청 설정 저장 시 더 이상 `advancedFeatureConfig`를 지우지 않도록 수정했다.

### 3. 키 안내 개선
*   `신청 복호화 키`는 기존 신청 본문 열람용이라는 점을 화면에 명시했다.
*   `새 키 생성`은 예전 신청을 열어주지 않고 앞으로 들어올 신청부터 적용된다는 점을 설명했다.
*   `라이선스 서명 키`도 기존 발급 체계를 유지하려면 원래 개인키를 넣어야 한다는 점을 표시했다.

### 4. 관리자 페이지 흐름 재배치
*   상단에 `빠른 이동` 카드를 추가했다.
*   기존 DOM을 그대로 이동하는 방식으로 `구독 운영` 카드와 `문구/후원 관리` 카드를 동적으로 구성했다.
*   `화면 전체 저장` 버튼은 현재 화면 기준으로 공지, 인기글 기준, 기본 설정, 고급 구독, 문구/후원, 수동 신청 설정까지 한 번에 저장하도록 넓혔다.

### 5. Firebase 규칙 보완
*   `stats/live_peak_daily` 읽기/쓰기 규칙을 추가했다.
*   관리자 저장용 `config/advancedFeatureConfig` 쓰기 규칙을 유지했다.
