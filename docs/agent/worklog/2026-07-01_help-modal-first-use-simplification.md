# 2026-07-01 사용 안내 모달 초보자 흐름 단순화

## 요청
- 사용 안내 페이지의 설명이 복잡하고 처음 쓰는 사람이 핵심 사용 방식을 알아보기 어렵다는 피드백이 있었다.
- 예시 화면에서는 개발자 공지와 기능 설명이 먼저 크게 보여 실제 사용 순서가 묻히는 문제가 있었다.

## 실행
- `index.html`의 사용 안내 모달 첫 화면을 3단계 핵심 흐름으로 재구성했다.
  - 문제 풀기
  - OMR에 답 체크
  - 정답 입력 후 채점
- 개발자 공지는 모달 맨 위에서 제거하고 `공지와 업데이트 보기` 접힘 영역으로 이동했다.
- 주요 기능 도식은 어두운 복잡한 다이어그램 대신 밝은 카드형 `핵심 버튼 위치` 안내로 단순화했다.
- `site-text-config.js`에 새 안내 문구 기본값과 관리자 편집 카탈로그 항목을 추가했다.
- 기존 원격 `siteTextConfig`가 과거 코드 기본 문구를 그대로 갖고 있는 경우 새 기본값으로 migration되도록 legacy 목록을 보강했다.
- 공지 접힘 영역은 일반 버튼 토글로 바꿔 마우스 클릭과 키보드 Enter/Space가 같은 경로로 동작하게 했다.
- `main.css`, `site-text-config.js`, `main.js` 쿼리 버전을 갱신해 기존 캐시가 새 안내/토글 코드를 가리지 않게 했다.

## 검증
- 로컬 정적 서버: `http://127.0.0.1:8765/?dev=1`
- 데스크톱 1280x720:
  - 사용 안내 모달 열림 확인
  - 첫 화면에 3단계 핵심 안내 표시 확인
  - 가로 overflow 없음
  - 콘솔 error/warn 없음
- 모바일 390x844:
  - 3단계 카드가 1열로 접힘 확인
  - 안내 문구와 카드 텍스트 overflow 없음
  - 콘솔 error/warn 없음
- 공지 접힘 영역:
  - 공지가 있을 때만 접힘 영역 표시 확인
  - 버튼 클릭으로 열림 상태 전환 확인

## 운영 메모
- 운영 RTDB `config/siteTextConfig` 값은 사용자 승인 없이 직접 수정하지 않았다.
- 운영 반영 시에는 관리자 페이지 또는 별도 승인된 절차로 `config/siteTextConfig` 동기화 여부를 함께 확인해야 한다.

## 추가 반영: 팝업 전환 강조와 안내 재단순화
- 왼쪽 사이드바의 팝업 버튼을 제거하고, 상단 타이머 오른쪽에 `먼저 팝업으로 열기` 버튼을 배치했다.
- 일반 화면에서만 팝업 전환 CTA가 보이고, 팝업 모드에서는 숨겨지도록 유지했다.
- 버튼과 짧은 안내 문구에 주기적 pulse 효과를 적용해 사용자가 먼저 누르도록 유도했다.
- 사용 안내 모달은 예시 이미지, 참고 설명, 기능 도식을 제거하고 네 단계만 남겼다.
  - 팝업으로 열기
  - 문제 풀기
  - OMR 체크
  - 채점
- 메모장/그림판은 탭으로 전환해 쓰며 다음 문항에서 자동 초기화된다는 설명만 남겼다.
- 계산기 설명은 제거했다.

## 추가 반영: 브라우저 주석 25건 수정
- 상단 팝업 CTA 문구에서 `먼저`를 제거해 `팝업으로 열기`로 변경했다.
- 팝업 버튼 주변에 주기적으로 나타났다 사라지는 상하좌우 화살표 강조 애니메이션을 추가했다.
- 안내 모달의 불필요한 첫 문장을 제거하고, 네 단계 문구를 사용자가 지적한 표현으로 정리했다.
  - `전환하여 사용합니다`
  - `연습 OMR을 눌러 답을 선택합니다`
  - `문항을 건너뛰기도 가능합니다`
  - `OMR 하단 정답 입력`
- 설정의 `연습 모드` 노출은 `풀이 방식` / `자유 풀이` 중심으로 바꿔 의미가 더 분명하게 보이도록 했다.
- 고급 모드 안내의 로그인 제목을 `구독 신청 때 사용한 ID와 PW로 로그인`으로 바꿨다.
- 고급 기능 안내의 긴 설명과 `세부 설명은 접어서 확인` 문구를 줄이고, 접힘 버튼은 `복기와 반복 연습이 편해지는 기능 보기`로 바꿨다.
- 구독 안내 제목을 `구독 요금제`로 바꾸고, 이용권 카드 아래에 `후원 페이지 열기` 버튼을 배치했다.
- 구독 요금제를 4종으로 재구성했다.
  - 3일권 2,900원
  - 7일권 3,900원
  - 14일권 5,900원
  - 30일권 9,900원
- 관리자 페이지의 수동 구독 설정에도 3일/7일/14일/30일권 가격 입력을 반영했다.
- 기존 원격 설정에 남아 있는 예전 7일/14일 가격과 14일권 강조 문구가 새 기본값을 덮지 않도록 보정했다.
- 신청 폼 라벨을 정리했다.
  - `투네이션 이름` -> `후원 닉네임`
  - `이용 시작일` -> `원하는 이용 시작일`
  - `로그인 ID` placeholder -> `원하는 이메일 주소`
  - `이메일(신청자)` -> `알림을 받을 이메일`
  - `비밀번호` -> `구독 조회 및 로그인에 사용할 비밀번호`
  - `비밀번호 규칙 보기` 접힘 항목 제거
- 고급 구독 로그인 후 닉네임 기반 환영 멘트를 세션당 1회 표시하도록 `main.js`에 추가했다.
- Firebase Functions 승인 경로에 신청자 승인 이메일 발송 helper를 추가했다. 비밀번호는 포함하지 않고 이용권, 로그인 ID, 만료일, 문의 이메일만 보낸다.
- 자동 발급 파이프라인 기본 요금제와 예시 설정을 새 요금제로 갱신했다.
