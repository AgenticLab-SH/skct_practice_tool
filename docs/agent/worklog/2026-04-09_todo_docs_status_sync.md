## 왜 정리가 필요했는가
- 기능 구현과 운영 반영은 상당수 끝났는데, TODO 문서 일부에는 예전 시점의 `배포 대기`, `rules 배포 필요`, `운영 반영 후 확인` 표현이 남아 있었습니다.
- 이렇게 되면 실제로 안 끝난 일이 있는 것처럼 보이거나, 다음 작업자가 같은 항목을 다시 착수할 위험이 있습니다.

## 이번에 맞춘 기준
- `10_부분해결`, `20_미해결`, `30_추가개선` 문서는 구현 완료 문서로 취급합니다.

- `docs/TODO/TODO.md`, `docs/TODO/TODO1.md`는 상태 문서가 아니라 당시 사용자 피드백 원본 메모로 유지합니다.
