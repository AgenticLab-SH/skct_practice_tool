# 고급 팝업 기본 폭 보정 및 안내 흐름 재정리 배포
작성일시: 2026-04-10 01:52:37 +09:00

## 1. 요청 요약

- 사용자가 `기본으로 열리는 팝업 크기 기준`에서 잘리는 부분이 없게 다시 맞춰 달라고 요청했습니다.
- 고급 모드 전체를 점검해 OMR 입력칸, 하단 버튼, 고급 안내/고급 활용 모달이 정상적으로 보이게 해 달라고 요청했습니다.
- 일반 모드와 고급 모드의 버튼 내부 설명이 서로 덜 겹치고, 더 짧고 명확하게 이어지도록 다듬어 달라고 요청했습니다.
- 수정 후 바로 공개 배포까지 진행해 달라고 요청했습니다.

## 2. 수행 내용

### 2.1 팝업 기본 폭과 OMR 가시성 보정

- `main.js`에서 팝업 기본값을 현재 운영 기준(`0.305 / 0.98 / 0.695 / 0`, `OMR 0.31`)과 맞췄습니다.
- 팝업 창 계산에서 최소 폭을 `520px`로 보정해, 작은 화면에서도 지나치게 좁은 팝업이 열리지 않게 했습니다.
- 팝업 모드에서는 OMR 최소 폭을 `150px`까지 보장하도록 조정했습니다.
- `main.css`에 팝업 전용 OMR 버튼/패널 밀도 조정을 넣어, 좁은 폭에서도 입력칸과 하단 버튼이 함께 보이도록 맞췄습니다.

### 2.2 일반 도움말 구조 재정리

- `index.html`, `main.css`, `site-text-config.js`에서 일반 도움말을 `일반 모드 기준`으로 다시 정리했습니다.
- 일반 도움말은 고급 기능 상세 설명을 줄이고, `고급 안내`와 `고급 활용`로 역할을 분리해 연결하도록 바꿨습니다.
- 참고사항, 사이드바 설명, 기본 흐름 문구를 짧게 줄이고 중복 설명을 제거했습니다.
- 도움말 안의 예시 카드와 기능 블록은 좁은 폭에서 1열로 자연스럽게 접히도록 반응형 구조를 넣었습니다.

### 2.3 고급 안내 / 고급 활용 역할 분리

- `고급 안내` 모달은 `신청, 승인 확인, 진입`만 설명하도록 다시 구성했습니다.
- `고급 활용` 모달은 `이미 들어온 뒤 어디를 누르는지`만 설명하도록 다시 구성했습니다.
- 고급 안내의 로그인 안내, 신청 순서, 신청서 설명, 상태 조회 설명을 짧고 명확한 문구로 다시 썼습니다.
- 고급 활용은 신청 절차 반복을 빼고, `상단 상태 바`, `OMR 아래 복기 버튼`, `자료 보관함 위치`만 빠르게 설명하도록 줄였습니다.
- 두 모달 모두 `500px 안팎 폭`과 `popup-mode`에서 2열 레이아웃이 남지 않도록 1열 반응형 클래스를 추가했습니다.

### 2.4 운영 문구 우선순위 보강

- 실제 화면에서 Firebase `config/siteTextConfig`가 예전 문구를 다시 덮는 문제를 막기 위해 `site-text-config.js` 기본값과 `LEGACY_SITE_TEXT_DEFAULTS`를 함께 갱신했습니다.
- 현재 운영에 저장된 도움말/고급 안내/고급 활용/고급 진입 상태 문구를 legacy 값으로 포함해, 새 배포 후에는 새 기본 문구로 자동 이행되도록 맞췄습니다.
- 특히 `messages.advancedAvailable`를 짧게 줄여, 고급 안내 상단 로그인 설명이 다시 길어지지 않게 했습니다.

### 2.5 자산 버전 갱신 및 공개 배포

- `build-info.js`, `index.html`, `admin.html`, `main.js`의 버전 문자열을 `v2026.04.10.0149`, `202604100149`로 올렸습니다.
- `scripts/export_public_clean.ps1`로 공개 배포 번들을 다시 만들었습니다.
