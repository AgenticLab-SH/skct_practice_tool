## 계산기 표시 구조 재조정
- 사용자 요청
  - 참고사진처럼 마지막 줄은 항상 현재 값 또는 계산 결과여야 함
  - 이전 값에서 바로 다음 연산으로 이어서 계산할 수 있어야 함
  - 진행 중인 수식은 위쪽 줄에 남고, 마지막 줄은 값 중심으로 보여야 함
- 수정 내용
  - 현재 줄 표시를 `수식 전체`에서 `현재 값/결과값` 표시로 변경
  - 진행 중인 식은 별도 `pending` 줄로 위쪽 이력 영역에 표시
  - 새 연산자를 눌러 연속 계산할 때도 직전 식이 위쪽 이력으로 누적되도록 수정
  - `=` 저장 형식도 `수식 = 결과` 한 줄이 아니라 `위쪽 수식 + 마지막 줄 결과` 구조에 맞게 조정
  - `SQRT`도 계산 결과는 마지막 줄에 두고, 적용식은 위쪽 이력에 남기도록 보정
 - 검증
   - `node --check main.js` 통과
   - `node --check staging/site/assets/scripts/app.bundle.js` 통과
   - 계산 시나리오 확인:
     - `7 + 8 =` -> 위줄 `7 + 8`, 마지막 줄 `15`
     - 이어서 `× 2` 입력 중 -> 위줄 `7 + 8`, `15 × 2`, 마지막 줄 `2`
     - 다시 `=` -> 위줄 `7 + 8`, `15 × 2`, 마지막 줄 `30`

## 계산기 우선순위 평가 전환
- 사용자 요청
  - 연산자 버튼만 눌러서는 바로 계산되지 않아야 함
  - `=`를 눌렀을 때만 계산되어야 함
  - 일반 계산기처럼 `*`, `/` 우선순위를 지켜야 함
  - 운영 배포까지 진행
- 수정 내용

  - `=` 입력 시에만 전체 식을 평가하도록 변경
  - 식 평가는 `*`, `/` 선처리 후 `+`, `-`를 처리하는 2단계 계산으로 구현
  - 연속 계산 시에도 결과값에서 새 식을 시작하도록 유지
