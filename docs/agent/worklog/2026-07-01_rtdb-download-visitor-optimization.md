# 2026-07-01 RTDB 다운로드 및 접속자 집계 최적화

## 요청
- Firebase Realtime Database 다운로드 사용량이 과도해질 수 있는 지점을 줄인다.
- 누적 접속자 집계 로직을 더 정확하게 점검한다.
- 일반 화면의 `자료` 버튼이 고급 자료 보관함과 혼동되는지 확인한다.
- Google 관리자 로그인이 성공했는지 알 수 없던 상태를 개선한다.

## 처리
- 관리자 Google 로그인은 popup 로그인을 먼저 시도하고, popup이 막히는 경우 redirect로 fallback하도록 변경했다.
- 로그인 실패 시 선택된 Google 이메일과 허용 관리자 이메일을 화면에 남긴다.
- 방문자 집계 날짜 기준을 브라우저 로컬 날짜에서 KST 날짜로 고정했다.
- 방문자 카운터는 DB transaction 성공 후에만 오늘 방문 처리 localStorage를 저장하도록 변경했다.
- 공개 통계는 `daily_visits.json` 전체가 아니라 최근 7일 경로만 개별 조회하도록 변경했다.
- 관리자 통계도 전체 `daily_visits`가 아니라 오늘 KST 경로만 조회하도록 변경했다.
- 커뮤니티 게시글 조회는 전체 `posts` 대신 `timestamp` 기준 최근 120개로 제한했다.
- 댓글 조회는 게시글별 최근 80개로 제한했다.
- `posts`와 `replies/<postId>`에 `timestamp` index를 추가했다.
- 일반 화면에 보이던 외부 테스트 안내 버튼 라벨을 `자료`에서 `테스트`로 바꿨다. 고급 자료 보관함 버튼은 기존처럼 `advanced-only`로 유지된다.

## 검증
- `node --check community.js` 통과.
- `node --check functions/index.js` 통과.
- 로컬 서버에서 `index.html`에 `테스트` 라벨, KST 날짜 함수, 최근 7일 개별 조회 코드 반영 확인.

## 남은 주의점
- 관리자 백업 기능은 의도적으로 DB 전체를 읽으므로 자주 실행하면 다운로드가 커질 수 있다.
- 커뮤니티 글이 120개를 넘으면 오래된 글은 일반 화면 기본 목록에서 빠진다. 검색/페이지네이션이 필요해지면 별도 UI로 확장한다.
