# 2026-04-08 전체 작업 요약
작성일시: 2026-04-08 15:45:21 KST

2026-04-08에는 먼저 Firebase 방문자·실시간 접속 지표를 바탕으로 유료화 방향을 검토하면서, 정기구독보다 기간제 이용권이 현재 서비스 구조에 더 맞는다는 판단을 정리했습니다.
그다음 고급 구독 관리자 화면을 `외부 ID` 중심 구조에서 `ID + 비밀번호` 중심 구조로 단순화하고, 상태·구독 종류·사용자 식별·ID·비밀번호·만료일·메모만 보이도록 정리했습니다.
이 과정에서 영구이용권 규칙, 관리자 비밀번호 기반 비밀번호 확인·초기화, 만료일 정오 12시 기준 처리까지 운영 코드에 묶어 반영했습니다.
이후 투네이션 후원 뒤 운영자가 승인하는 수동 고급 이용권 신청 흐름을 새로 설계하고, 신청번호·조회 비밀번호 기반 조회, 브라우저 암호화 저장, 관리자 승인 UI, 전체 DB 백업·복원 안전장치까지 붙였습니다.
사용자 화면 쪽도 일반모드와 고급모드 안내 문구를 손봐서 `로그인 -> 기능 이해 -> 신청 -> 조회` 순서로 이해되도록 흐름을 다시 잡았습니다.
그 다음에는 운영자가 코드 수정 없이 문구를 바꿀 수 있도록 `site-text-config.js` 기반 텍스트 카탈로그와 관리자용 사이트 텍스트 편집기, iframe 실시간 미리보기, 클릭 선택 기능을 만들었습니다.
운영 페이지와 관리자 페이지가 같은 `config/siteTextConfig`를 읽도록 연결해서, 미리보기는 즉시 반영되고 실제 운영값은 저장 버튼을 눌렀을 때만 Firebase RTDB에 저장되게 설계했습니다.
마지막으로 사용자 피드백에 따라 사이트 텍스트 편집기 좌측 목록이 우측 미리보기 높이에 끌려가지 않도록 `sticky + max-height + 내부 overflow` 구조로 고쳐 독립 스크롤되게 만들고 브라우저로 검증했습니다.
이 모든 변경은 오늘 `main`에 순차적으로 반영했고, 최종적으로 GitHub Pages 운영 주소 응답과 원격 `main` HEAD를 확인해 실제 사이트 반영까지 마무리했습니다.
