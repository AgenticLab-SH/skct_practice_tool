# 공지 저장 실패 원인 노출 보강
작성일시: 2026-04-09 17:12:00 KST

## 요청 요약
- 관리자 페이지 `공지 관리`에서 사이드바 공지를 수정하고 저장했지만 실제 운영 반영이 되지 않는 이유를 확인해 달라는 요청.

## 확인 결과
- 운영 DB `config/notice_sidebar`에는 여전히 아래 예전 문구만 저장돼 있었다.
  - `메모장 일부분이 안 그려지는 문제를 개선하였습니다...`
- 사용자가 적었다고 한 추가 문구
  - `팝업 연습 버튼을 눌러서 사용해주세요!`
  는 운영 DB의 `notice_sidebar`, `notice_help`, `notice` 어디에도 들어가지 않았다.
- 즉 이번 문제는 메인 페이지 표시 문제가 아니라 관리자 저장 자체가 실패한 상태였다.

## 원인 판단
- Firebase RTDB rules상 `config` 쓰기는 `auth != null` 조건이다.
- 따라서 로그인 세션이 끊겼거나 권한이 없으면 저장 시 `Permission denied`가 발생한다.
- 기존 관리자 페이지는 `공지 저장` 같은 개별 저장 버튼에서 이 실패를 버튼 클릭 지점에서 충분히 보여주지 못했다.

## 수정 내용
- `admin.html`
  - 저장 실패 메시지 포맷터 `formatAdminSaveError()` 추가
  - 개별 저장 버튼 공통 래퍼 `runAdminSaveAction()` 추가
  - 권한 실패는 `새로고침 후 다시 로그인`, 네트워크 실패는 `연결 확인 후 재시도` 안내 토스트로 표시
- 적용 대상
  - 공지 저장
  - 인기글 기준 저장
  - 앱/링크 설정 저장
  - 타이머/레이아웃 저장
  - 후원/운영 문구 저장
  - 사이트 텍스트 저장
  - 후원 내역 저장
  - 수동 구독 설정 저장

## 사용자 안내
- 현재 운영 DB에는 사용자가 입력한 새 공지 문구가 아직 반영되지 않았다.
- 관리자 페이지를 새로고침하고 다시 로그인한 뒤 `사이드바 공지` 대상에서 다시 저장해야 한다.
- 이때 실패하면 토스트에 원인이 바로 표시된다.
