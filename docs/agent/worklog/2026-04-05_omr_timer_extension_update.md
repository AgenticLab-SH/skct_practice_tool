# 2026-04-05 OMR·타이머·무적모드 개선 작업 기록
작성일시: 2026-04-05 11:16:36 KST

## 사용자 요청
- OMR 각 번호 옆 `건너뛰기` 버튼 제거
- 과목별 상세 통계를 과목별 토글 접기 구조로 변경하고, 소요시간도 과목별로 묶기
- 무적모드를 테스트 기능으로 명시하고 ZIP에 개인환경 정보가 없는지 점검
- 정답 입력 모드 진입 시 전체 타이머와 가이드 타이머 자동 정지
- 타이머 전체시간 기본값을 `75`로 변경
- 관리자 페이지에서 `전체 시간` 저장이 실제 반영되지 않는 문제 수정
- 수정 후 GitHub 푸시

## 수행 내용
- `main.js`
  - 문항 행의 개별 `건너뛰기` 버튼 렌더링과 이벤트 제거
  - `stopTimer()` / `enterScoreMode()` 추가로 정답 입력 모드 진입 시 타이머 즉시 정지
  - 상세 통계를 과목별 `details` 묶음으로 재구성
  - 총시간 최소 강제 보정 제거
- `index.html`
  - 무적모드 버튼/모달에 `(테스트)` 표기 반영
  - 사용자 기본 총시간 표시와 설정 입력값을 `75`로 변경
- `admin.html`
  - 타이머 저장 보정 함수에서 `79분 이상` 강제 로직 제거
  - 관리자 기본 입력값을 `75`로 변경
- `chrome-extension/popup.html`
  - 확장 팝업에도 테스트 기능 안내 문구 반영
- `skct_extension.zip`
  - 수정된 확장 폴더 기준 재생성

## 검증 결과
- Playwright 로컬 테스트
  - 문항별 개별 스킵 버튼 미노출 확인
  - 타이머 실행 후 정답 입력 모드 전환 시 `▶ 시작 / 정지`로 복귀, 2초 후에도 시간 고정 확인
  - 가이드 타이머 숨김 확인
  - 과목별 상세 통계에 `details` 5개 생성, 모두 기본 접힘 상태 확인
  - 관리자 저장 동선으로 `75/15/1` 저장 후 프리뷰에서 `75:00` 반영 확인
- ZIP 점검
  - 포함 파일: `content.js`, `icon128.png`, `icon48.png`, `manifest.json`, `popup.html`
  - 절대경로, 사용자명, 토큰, 개인환경 문자열 검색 결과 없음

## 추가 후속 조치
- 원격 Firebase `config/timerDefaults`를 직접 조회해 실제 값이 `75/15/1`임을 확인
- 실제 웹에서 `79`가 계속 남는 원인을 `localStorage.skct_timer_cfg`의 예전 기본값 `79/15/1`로 확인

- 이후 사용자 수동 저장값은 `source: 'user'` 메타데이터와 함께 저장하도록 변경
