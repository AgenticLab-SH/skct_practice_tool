# 2026-04-08 개발자 페이지 비밀번호 확인
작성일시: 2026-04-08 16:34:06 +09:00

사용자 질문: `내가 개발자 페이지 비밀번호를 뭘 했더라?`

## 확인 결과
- `admin.html`의 관리자 로그인은 평문 비밀번호를 코드에 저장하지 않고, Firebase RTDB의 `config/adminHash`와 입력값의 SHA-256 해시를 비교하는 방식이다.
- 현재 운영 RTDB `config/adminHash`에는 해시값이 실제로 들어 있다. 즉, "아직 비밀번호가 비어 있어서 첫 입력값이 설정되는 상태"는 아니다.
- 저장된 평문 비밀번호는 저장소와 문서에서 찾지 못했다. 현재 확인 가능한 것은 해시 존재 여부뿐이다.

## 혼동하기 쉬운 값
- 문서와 코드에서 보이는 `0208`은 개발자 페이지(`admin.html`) 비밀번호가 아니다.
- `0208`은 고급 기능 인증 비밀번호 기본값으로 기록된 값이다.

## 근거 위치
- `admin.html:1143` 부근: `config/adminHash`를 읽고 SHA-256 비교
- `50_AGENT_LAST_WORK_REPORT.md`의 2026-04-06 기록: 고급 기능 기본 비밀번호 `0208`
- `docs/agent/worklog/2026-04-08_comprehensive_project_audit.md`: `config/adminHash`가 비인증 읽기 가능한 구조라는 점 기록
