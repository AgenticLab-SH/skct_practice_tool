# 로컬 관리자 계정 저장 및 로그인 확인
작성일시: 2026-04-10 10:04:36 +09:00

## 요청

- 관리자 계정을 로컬에 저장하고 Git 추적에서 제외한 뒤, 그 자격정보로 관리자 페이지 로그인을 시도해 달라는 요청이 있었습니다.

## 수행 내용

- `.gitignore`에 `private/`를 추가해 로컬 비밀 파일이 Git에 올라가지 않도록 했습니다.
- 로컬 전용 환경 파일 `private/admin.local.env`를 만들고, 관리자 이메일/비밀번호를 `SKCT_ADMIN_EMAIL`, `SKCT_ADMIN_PASSWORD` 환경변수 형태로 저장했습니다.
- 로컬 `http.server`로 `admin.html`을 열고, 저장한 자격정보로 실제 로그인 버튼을 눌러 Firebase Auth 로그인까지 확인했습니다.

## 결과

- 비밀 파일은 Git 추적 대상에서 제외된 상태입니다.
- 처음 전달받은 값으로는 실제 로그인이 실패했습니다.
- 이후 수정된 자격정보로 다시 확인했고, 로컬 `admin.html` 로그인에 성공했습니다.
- 로그인 후 관리자 대시보드, 구독 운영 탭, 신청 목록, 고급 구독 표가 정상 노출되는 것을 확인했습니다.
- 로그인 성공 직후 브라우저 콘솔 기준 추가 에러는 없었습니다.

## 판단

- 이번 턴에서 할 수 있는 로컬 저장과 Git 제외는 끝났습니다.
- 이제 이 PC에서는 저장한 로컬 비밀 파일을 기준으로 관리자 로그인 재사용이 가능합니다.
- 기능 문서 변경 없음.
