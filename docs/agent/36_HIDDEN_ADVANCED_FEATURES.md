# 숨김 고급 기능 설계 메모
작성일시: 2026-04-08 11:22:05 KST

이 문서는 일반 사용자 화면과 고급 사용자 화면을 분리하면서, 인증된 사용자만 고급 기능에 진입시키는 현재 설계를 정리한 문서입니다.

## 진입 방식
- 일반 화면 좌측 사이드바 `🔒 고급 안내` 버튼에서 고급 기능 설명과 비밀번호 입력 UI를 엽니다.
- 이미 이용권이 있는 사용자는 그 모달에서 바로 `ID + 비밀번호`를 입력해 고급 버전 팝업을 열 수 있습니다.
- 기존 숨김 진입도 유지합니다.
  - `통합 설정` 모달 제목 7회 연속 클릭
  - 1.8초 내 연속 입력 기준
  - 이 경로는 이스터에그 성격으로 남겨둡니다.

## 인증 방식
- 고급 기능은 별도 인증 팝업(`advanced-tools.html`)에서 `ID + 비밀번호`를 확인합니다.
- 일반 화면의 `고급 안내` 모달에서도 직접 인증할 수 있습니다.
- 허용 계정 목록은 개발자 페이지에서만 수정합니다.
- 실제 앱은 원격 설정 `advancedFeatureConfig.subscriptions`를 읽고, `passwordSalt + passwordHash` 기준으로 검증합니다.
- 운영에는 기존 평문 비밀번호 목록을 해시 구독 레코드로 마이그레이션했습니다.
- 인증 성공 시 같은 창의 이름을 실제 팝업 이름(`skct_popup_mode`, 스테이징은 `stg_skct_popup_mode`)으로 바꾸고, 실제 사용자 화면 `index.html?advanced=1`로 이동시킵니다.
- 고급 기능은 전환된 실제 사용자 팝업의 좌측 사이드바 `🔒 고급 기능` 버튼에서만 노출합니다.

## 일반 기능과 분리한 범위
- 일반 사용자 화면에는 `고급 안내`만 노출하고, 실제 확장 기능은 고급모드에서만 노출합니다.
- 문항별 상세 통계 텍스트 다운로드는 `advanced=1`로 열린 고급 모드 사용자 팝업의 전용 모달에서만 제공합니다.
- 구독/비밀번호 저장은 개발자 페이지에서만 제공합니다.
- 타이머/문항 건너뛰기/계산기 로직 자체는 실제 기능 오류 수정이므로 일반 화면에도 반영합니다.

## 최근 보강 사항
- `고급 안내` 모달에서 현재 사용 가능한 고급 이용권 수를 바로 보여줍니다.
- 인증된 브라우저는 일정 시간 동안 `바로 열기`로 재진입할 수 있습니다.
- 비밀번호 오입력이 누적되면 짧은 재시도 대기시간을 둡니다.
- 개발자 페이지는 textarea 대신 `고급 구독 관리 표`로 바뀌어, 상태/구독 종류/사용자 식별/ID/비밀번호/만료일/메모를 같이 관리합니다.

## 관리자 화면 용어 기준
- `사용자 식별`: 이름/이메일처럼 운영자가 누구 계정인지 확인하는 값
- `ID`: 사용자가 실제 로그인할 때 입력하는 값
- `비밀번호`: 저장 시 해시로 바뀌는 로그인 비밀번호
- `만료일`: 선택한 날짜의 정오 12:00(KST)까지 유효

자세한 운영 절차는 아래 문서를 우선 참고합니다.
- [30_ADVANCED_SUBSCRIPTION_ADMIN_GUIDE.md](/C:/dev/01_career/_assets/tools/skct_tool/docs/agent/30_ADVANCED_SUBSCRIPTION_ADMIN_GUIDE.md)

## 이번 턴에서 함께 정리한 핵심 로직
- 문항 건너뛰기 시 `questionSpentSec`가 즉시 0으로 초기화되도록 질문 전환 함수를 하나로 통합했습니다.
- 전체 시간은 설정값과 페이즈 합계가 어긋날 때 조기 종료가 나지 않도록 `max(입력 전체 시간, 과목+쉬는시간 합계)` 기준으로 보호합니다.
- 계산기는 `Ans = 결과값` 아카이브와 `수식 = 결과` 기록을 분리해, 최근 3개 이력 + 현재 줄 1개 구조를 유지합니다.

## 파일 분리 원칙
- 메인 화면 진입 제어와 런타임 데이터 브리지는 `main.js`, `staging/site/assets/scripts/app.bundle.js`에 둡니다.
- 인증 게이트 UI는 `advanced-tools.html`, `staging/site/advanced-tools.html`로 분리합니다.
- 실제 고급 기능 UI는 `index.html`, `staging/site/index.html`의 전용 모달 안에 둡니다.
- 인증 비밀번호 관리 UI는 `admin.html`, `staging/site/admin.html`에 둡니다.
- 사용자 요청과 실제 구현 내역은 작업 로그와 이 문서에 함께 기록합니다.
