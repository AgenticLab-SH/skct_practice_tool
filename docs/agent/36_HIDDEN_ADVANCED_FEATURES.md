# 숨김 고급 기능 설계 메모
작성일시: 2026-04-06 02:47:52 KST

이 문서는 일반 사용자 화면에 힌트를 남기지 않으면서, 인증된 사용자만 고급 기능에 진입시키는 현재 설계를 정리한 문서입니다.

## 진입 방식
- 일반 설정 버튼을 눌러 여는 `통합 설정` 모달의 제목을 연속 입력했을 때만 고급 기능 팝업을 엽니다.
- 일반 화면에는 별도 버튼, 문구, 안내를 두지 않습니다.
- 숨김 진입은 `7회 연속 클릭`, `1.8초 내 연속 입력` 기준으로 처리합니다.

## 인증 방식
- 고급 기능은 별도 인증 팝업(`advanced-tools.html`)에서 비밀번호만 확인합니다.
- 팝업 진입 직후 비밀번호를 요구합니다.
- 허용 비밀번호 목록은 브라우저 저장소에 배열로 보관합니다.
- 기본값은 `0208` 1개만 넣습니다.
- 인증 성공 시 같은 창의 이름을 실제 팝업 이름(`skct_popup_mode`, 스테이징은 `stg_skct_popup_mode`)으로 바꾸고, 실제 사용자 화면 `index.html?advanced=1`로 이동시킵니다.
- 고급 기능은 전환된 실제 사용자 팝업의 설정 모달 안에서만 노출합니다.

## 일반 기능과 분리한 범위
- 일반 사용자 화면에는 고급 기능 버튼을 노출하지 않습니다.
- 문항별 상세 통계 텍스트 다운로드와 비밀번호 목록 저장은 `advanced=1`로 열린 고급 모드 사용자 팝업의 설정 모달에서만 제공합니다.
- 타이머/문항 건너뛰기/계산기 로직 자체는 실제 기능 오류 수정이므로 일반 화면에도 반영합니다.

## 이번 턴에서 함께 정리한 핵심 로직
- 문항 건너뛰기 시 `questionSpentSec`가 즉시 0으로 초기화되도록 질문 전환 함수를 하나로 통합했습니다.
- 전체 시간은 설정값과 페이즈 합계가 어긋날 때 조기 종료가 나지 않도록 `max(입력 전체 시간, 과목+쉬는시간 합계)` 기준으로 보호합니다.
- 계산기는 `Ans = 결과값` 아카이브와 `수식 = 결과` 기록을 분리해, 최근 3개 이력 + 현재 줄 1개 구조를 유지합니다.

## 파일 분리 원칙
- 메인 화면 진입 제어와 런타임 데이터 브리지는 `main.js`, `staging/site/assets/scripts/app.bundle.js`에 둡니다.
- 인증 게이트 UI는 `advanced-tools.html`, `staging/site/advanced-tools.html`로 분리합니다.
- 실제 고급 기능 UI는 `index.html`, `staging/site/index.html`의 설정 모달 안에 둡니다.
- 사용자 요청과 실제 구현 내역은 작업 로그와 이 문서에 함께 기록합니다.
