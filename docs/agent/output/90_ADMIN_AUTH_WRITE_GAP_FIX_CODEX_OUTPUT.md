# 관리자 인증-쓰기 간극 수정 결과
작성일시: 2026-04-09 17:36:00 KST

## 확인한 사실
- 운영 DB `config/notice_sidebar`에는 아직 예전 공지 문구만 저장돼 있습니다.
- 사용자가 입력한 `팝업 연습 버튼을 눌러서 사용해주세요!` 문장은 운영 DB에 아직 없습니다.
- 즉 이번 시점의 문제는 공지 렌더링이 아니라 관리자 저장 자체가 실제 DB에 쓰이지 않은 상태였습니다.

## 핵심 원인
- `2026-04-08 18:19:51 KST`의 `aa6ccc20`에서 관리자 인증이 Firebase Auth 기반으로 바뀌었습니다.
- 그 뒤 관리자 화면은 `onAuthStateChanged()`에서 사용자 객체만 있으면 대시보드에 들어갔습니다.
- 하지만 실제 RTDB 쓰기 직전에는 토큰 유효성을 다시 확인하지 않아, 화면상 로그인 상태와 실제 DB 쓰기 가능 상태가 어긋날 수 있었습니다.

## 이번 수정
- 운영 반영 커밋: `fbec216`
- 관리자 페이지에 아래 공통 보강을 추가했습니다.
  - 대시보드 진입 전 토큰 확인
  - 모든 관리자 쓰기 전 토큰 확인
  - `Permission denied`면 강제 토큰 갱신 후 1회 자동 재시도
  - 그래도 실패하면 즉시 토스트로 실패 이유 안내

## 운영 확인
- 라이브 `admin.html`에 `ensureAdminWriteSession`, `runAdminDatabaseWrite`, `user.getIdToken(forceRefresh)` 포함 확인
- 라이브 관리자 페이지 로그인 화면 로드 및 콘솔 에러 0건 확인

## 사용자가 지금 할 일
1. 관리자 페이지를 새로고침합니다.
2. 다시 로그인합니다.
3. `사이드바 공지` 대상인지 확인합니다.
4. 공지 문구를 다시 저장합니다.

## 기대 결과
- 이제는
  - 저장이 정상 반영되거나
  - 토큰/권한 문제면 자동 재시도 후 명확한 오류 메시지가 바로 보입니다.
