# 공지 저장 실패 원인 확인 및 관리자 안내 보강
작성일시: 2026-04-09 17:18:00 KST

## 현재 상태
- 운영 DB `config/notice_sidebar`에는 아직 예전 문구만 저장돼 있습니다.
- 사용자가 입력했다고 한 추가 문구 `팝업 연습 버튼을 눌러서 사용해주세요!` 는 운영 DB에 반영되지 않았습니다.

## 왜 안 바뀌었는가
- 이번 문제는 공개 페이지가 새 공지를 못 읽는 문제가 아니라, 관리자 저장 자체가 운영 DB에 쓰이지 않은 문제입니다.
- RTDB rules상 `config` 쓰기는 로그인된 관리자(`auth != null`)만 가능합니다.
- 로그인 세션이 끊겼거나 권한 문제가 있으면 `Permission denied`가 나는데, 기존 관리자 페이지는 개별 저장 버튼에서 이 실패를 충분히 알려주지 못했습니다.

## 이번에 반영한 수정
- 운영 `admin.html`에 저장 실패 공통 처리 로직을 추가했습니다.
- 이제 `공지 저장` 등 개별 저장 버튼이 실패하면 바로 토스트로 아래 같은 안내를 보여줍니다.
  - 권한 실패: 새로고침 후 다시 로그인
  - 네트워크 실패: 연결 확인 후 재시도

## 사용자가 지금 해야 할 일
1. 관리자 페이지를 새로고침합니다.
2. 다시 로그인합니다.
3. `공지 관리`에서 대상이 `사이드바 공지`인지 확인합니다.
4. 아래 문구를 다시 저장합니다.

```text
메모장 일부분이 안 그려지는 문제를 개선하였습니다. 컨트롤 + f5 를 눌러서 페이지 강력 새로고침 후에 사용해주세요. 그래도 문제가 발생할 시에 게시판에 남겨주세요!
팝업 연습 버튼을 눌러서 사용해주세요!
```
