# 구독 로그인 ID·이메일 정리 결과
작성일시: 2026-04-09 08:41:44 KST

## 변경 요약
- 공개 로그인은 이제 `신청 이메일 또는 로그인 ID + 비밀번호`만 사용합니다.
- `신청번호`는 공개 로그인과 신청 조회에서 사용하지 않도록 정리했습니다.
- 관리자가 직접 추가한 `ID만 있는 계정`은 기존처럼 `로그인 ID + 비밀번호`로 계속 로그인됩니다.
- 로그인 ID는 일반 문자열 ID와 이메일 형식 둘 다 사용할 수 있게 문구와 로직을 맞췄습니다.

## 확인 결과
- `REQ-...` 형태 값은 로그인으로 통과하지 않습니다.
- 이메일이 아닌 `ID-only` 계정은 계정 로그인 경로로 정상 통과합니다.
- 이메일 형식 로그인 ID도 신청 조회 실패 시 계정 로그인 경로로 자동 fallback 되어 정상 통과합니다.
- 메인 고급 안내, 팝업 인증 화면, 관리자 설명 문구를 새 정책에 맞게 정리했습니다.

## 검증
- `node --check main.js`
- `node --check site-text-config.js`
- 로컬 서버 `http://127.0.0.1:8124/`에서 고급 안내 모달과 팝업 인증 문구 확인
- Playwright mock 검증으로 `신청번호 차단`, `ID-only 허용`, `이메일형 ID 허용` 확인
