## 핵심 원인
- 로그인 자체는 성공했지만, `?advanced=1` 페이지가 열리는 순간 원격 라이선스 공개키가 아직 오기 전이라 저장된 고급 라이선스를 잘못 지워 버리고 있었습니다.
- 그래서 새 페이지는 “라이선스 없음” 상태로 시작했고, 일반 모드로 되돌아갔습니다.

- 공개키 또는 검증 함수가 준비되기 전에는 저장된 고급 라이선스를 삭제하지 않도록 바꿨습니다.
- `index.html`
- 실제 원격 `manualSubscriptionConfig` 적용 시점에만 라이선스 동기화가 돌도록 호출 인자를 추가했습니다.
