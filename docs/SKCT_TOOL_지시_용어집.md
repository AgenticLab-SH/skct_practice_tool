# SKCT Tool 지시 용어집
작성일시: 2026-04-06 10:39:06 KST

이 문서는 사용자가 저에게 지시할 때 쓰는 용어와 명령 방식을 고정하기 위한 기준입니다.  
앞으로는 아래 표현을 기본 표준으로 사용하면 가장 빠르고 오해 없이 작업할 수 있습니다.

## 1. 환경 용어 표준

| 기존에 섞여 쓰던 말 | 앞으로 쓸 표준 용어 | 의미 |
| --- | --- | --- |
| normal 모드, normal 버전 | 일반모드 | 일반 사용자용 화면 |
| 고급버전, advanced, 숨김 버전 | 고급모드 | 숨김 인증 후 진입하는 고급 사용자용 화면 |
| 테스트사이트, 테스트 페이지 | 스테이징 | 운영 반영 전 검증용 원격 화면 |
| 실서버, 실제 서버, 라이브 | 운영 | 실제 사용자들이 쓰는 서버/페이지 |
| 로컬, 내 PC 테스트 | 로컬 | 현재 작업 PC에서만 확인되는 상태 |
| 개발자 페이지, 관리자 페이지 | 관리자 페이지 | 설정 저장/기본값 관리 화면 |

## 2. 권장 표현

- `실서버` 대신 `운영`을 권장합니다.
- 이유:
  - 짧고 명확합니다.
  - `운영 반영`, `운영 확인`, `운영 롤백`처럼 조합이 쉽습니다.
  - `서버`라는 표현이 실제로는 GitHub Pages 정적 페이지인지, Firebase 설정인지 헷갈릴 수 있는데 `운영`은 범위를 더 잘 묶어줍니다.

## 3. 작업 단계 표준

앞으로는 아래 4단계 표현을 기본으로 쓰면 됩니다.

1. `로컬 작업`
2. `스테이징 반영`
3. `스테이징 확인`
4. `운영 반영`

예시:
- `이건 먼저 로컬에서 수정해`
- `스테이징에 올려`
- `스테이징 확인 끝나면 운영 반영해`

## 4. 자주 쓰는 지시 표현 표준

### 4.1 수정 범위
- `일반모드만 수정해`
- `고급모드만 수정해`
- `일반모드와 고급모드 둘 다 수정해`
- `관리자 페이지만 수정해`
- `팝업 화면만 수정해`
- `계산기만 건드려`

### 4.2 반영 범위
- `로컬만 반영해`
- `스테이징까지만 반영해`
- `운영까지 반영해`
- `운영은 건드리지 마`

### 4.3 검증 방식
- `로컬 검증까지 해`
- `스테이징에서 확인 가능하게 해`
- `운영 반영 전에 회귀 체크해`
- `다른 기능 영향 없는지 확인해`

### 4.4 데이터/설정 관련
- `기본값으로 저장해`
- `운영 기본값은 바꾸지 마`
- `스테이징 기본값만 바꿔`
- `관리자 페이지에서만 저장 가능하게 해`
- `Firebase 저장 경로도 같이 확인해`

### 4.5 배포/이식 관련
- `스테이징에서 승인되면 운영에 이식해`
- `지금 수정한 걸 운영에도 같은 방식으로 반영해`
- `운영 코드는 나중에, 먼저 스테이징만`

## 5. 제가 해석하는 기준

앞으로 사용자가 아래처럼 말하면 저는 이렇게 해석합니다.

- `반영해`
  - 문맥상 마지막으로 이야기한 대상 환경에 반영
- `올려`
  - 원격 반영 또는 배포
- `이식해`
  - 이미 한 환경에서 만든 변경을 다른 환경에 같은 방식으로 복사 반영
- `체크해`
  - 화면/로직 검증
- `확인해`
  - 현재 상태 점검
- `기본값으로 해`
  - 코드 fallback 값 + 저장된 관리자 기본값 기준으로 맞춤

## 6. 애매한 표현 대신 권장 표현

아래 표현은 헷갈릴 수 있으니 가능하면 권장 표현으로 바꾸는 것이 좋습니다.

| 애매한 표현 | 권장 표현 |
| --- | --- |
| 실서버 | 운영 |
| 테스트 | 스테이징 또는 로컬 |
| 버전 | 일반모드 / 고급모드 / 스테이징 / 운영 중 하나로 구체화 |
| 저장해 | 스테이징 기본값 저장 / 운영 기본값 저장처럼 대상 명시 |
| 적용해 | 로컬 적용 / 스테이징 반영 / 운영 반영처럼 환경 명시 |

## 7. 가장 추천하는 짧은 명령 패턴

아래 형식으로 말하면 가장 빠르게 처리할 수 있습니다.

`[대상] + [범위] + [환경] + [검증 여부]`

예시:
- `계산기만 일반모드에서 수정하고 스테이징 반영 후 검증해`
- `관리자 페이지 기본값 저장 로직만 고급모드 기준으로 바꾸고 운영 반영해`
- `팝업 UI를 일반모드/고급모드 둘 다 맞추고 스테이징에서 먼저 확인하게 해`

## 8. 앞으로 바로 써도 되는 추천 용어 세트

- `normal 모드` -> `일반모드`
- `고급버전` -> `고급모드`
- `테스트사이트` -> `스테이징`
- `실서버` -> `운영`
- `개발자페이지` -> `관리자 페이지`
- `기본 저장값` -> `기본값`
- `실제 적용` -> `운영 반영`
- `복붙해서 옮기기` -> `이식`

## 9. 한 줄 요약

- `일반모드`
- `고급모드`
- `로컬`
- `스테이징`
